Déployer sur Railway
Plan démarrage gratuit · Conteneurs toujours actifs · Déploiements auto depuis GitHub · Sans carte bancaire.
Open Railway →
1
Créer un compte Railway gratuit
Go to railway.app and sign up with GitHub. It's free — no credit card needed to start.
2
Créer un nouveau projet → Déployer depuis GitHub
Click + New Project → Deploy from GitHub repo.
If you haven't pushed your bot to GitHub yet, first push it:
Terminal commands
cd your-bot-folder/
git init
git add .
git commit -m "Initial bot setup"
# Then create a GitHub repo and push
git remote add origin https://github.com/you/your-bot.git
git push -u origin main
3
Ajouter vos variables d'environnement
In your Railway project → Variables tab → add these:
Required environment variables
SESSION_IDYour SIGMA-MDX~ session ID
BOT_NAMEYour bot name
OWNER_NUMBER+32466304227 (with country code)
PREFIX. (or your chosen prefix)
MODEpublic / private / inbox / group
NODE_ENVproduction
4
Déployer & vérifier
Click Deploy. Railway will install dependencies and start your bot automatically. Check the Logs tab — you should see WhatsApp connection established within 60 seconds.
5
Maintenir actif (optionnel)
Free plan includes 500 hours/month. To stay always-on, upgrade to the Hobby plan ($5/mo) or use Railway's sleep prevention setting in the service configuration.
💡 Conseils Railway
  • Railway automatically uses your package.json start script (node index.js)
  • The railway.toml in your ZIP is already configured — no extra setup needed
  • Use Redeploy button after changing env vars to apply them
  • Railway's free tier restarts after 1h of inactivity — normal for the first deploy

Déployer sur Render
Services web gratuits · Déploiements auto depuis GitHub · 750 heures gratuites/mois · Sans carte bancaire.
Open Render →
1
S'inscrire & connecter GitHub
Go to render.com → Sign up with GitHub → authorize Render to access your repositories.
2
Pousser votre bot sur GitHub
Upload your bot ZIP to a GitHub repo (public or private). Render supports both.
git init && git add . && git commit -m "Bot setup"
git remote add origin https://github.com/you/bot.git
git push -u origin main
3
Créer un nouveau Web Service
Click New → Web Service → select your repo.
Set Environment to Node, Build Command to npm install, Start Command to npm start.
4
Ajouter des variables d'environnement
In Environment tab → Add Environment Variable:
Required variables
SESSION_IDYour SIGMA-MDX~ session ID
BOT_NAMEYour bot name
OWNER_NUMBER+32466304227
PREFIX.
MODEpublic
5
Déployer
Click Create Web Service. Render will build & deploy automatically. Check logs to confirm the bot starts up correctly.
💡 Conseils Render
  • Free tier spins down after 15 min of inactivity — use a cron ping (UptimeRobot) to keep it awake
  • The render.yaml in your ZIP is pre-configured for zero-setup deploy
  • Enable Auto-Deploy to redeploy on every GitHub push

Déployer sur Koyeb
Niveau gratuit généreux · Conteneurs toujours actifs · Déploiement global · Pas de mise en veille.
Open Koyeb →
1
Créer un compte Koyeb
Go to app.koyeb.com → sign up for free. No credit card required for the Starter plan.
2
Créer une nouvelle App depuis GitHub
Click Create App → select GitHub as the deployment source → authorize and select your repository.
3
Configurer le service
Set Runtime to Node.js.
Set Run command to npm start.
Set Instance type to eco (free).
4
Ajouter des variables d'environnement
In the Environment variables section:
Required variables
SESSION_IDYour SIGMA-MDX~ session ID
BOT_NAMEYour bot name
OWNER_NUMBER+32466304227
PREFIX.
MODEpublic
5
Déployer & vérifier
Click Deploy. Koyeb's free plan includes always-on containers — no sleep! Check the Logs to confirm the bot connects to WhatsApp.
💡 Conseils Koyeb
  • Best free option — Koyeb's free tier does NOT sleep like Render's
  • The koyeb.yaml in your ZIP is pre-configured
  • Set Health check to disabled if your bot doesn't expose an HTTP port

Déployer sur Heroku
PaaS fiable · Dyno Eco à 5$/mois · Intégration GitHub facile · Infrastructure de production.
Open Heroku →
1
Créer un compte Heroku & installer le CLI
Sign up at signup.heroku.com. Then install the Heroku CLI on your machine.
2
Créer une nouvelle app Heroku
heroku login
heroku create your-bot-name
git init && git add . && git commit -m "Bot setup"
heroku git:remote -a your-bot-name
git push heroku main
3
Définir les variables d'environnement
Via CLI or Heroku Dashboard → Settings → Config Vars:
Config Vars
SESSION_IDYour SIGMA-MDX~ session ID
BOT_NAMEYour bot name
OWNER_NUMBER+32466304227
PREFIX.
MODEpublic

Or via CLI: heroku config:set SESSION_ID=SIGMA-MDX~xxxx BOT_NAME=MyBot
4
Mettre à l'échelle le dyno
heroku ps:scale worker=1
Your Procfile (included in the ZIP) declares: worker: node index.js
5
Vérifier les logs
heroku logs --tail — watch for WhatsApp connection established to confirm the bot is running.
💡 Conseils Heroku
  • Use Eco dynos ($5/mo for 1000 hrs) — basic dynos sleep after 30min on free tier
  • The app.json and Procfile in your ZIP are pre-configured for Heroku
  • Add a PostgreSQL add-on if you want persistent database: heroku addons:create heroku-postgresql:mini
  • Use Heroku Scheduler add-on for cron jobs if needed
